

You raise a valid point. The teen account is linked to a parent's household, so the currency is inherited from the parent's profile setting. Adding a separate country/currency selector to the teen profile would be redundant and could create inconsistencies — the teen would see one currency while the parent sees another for the same household.

There is no need to implement this. The current architecture where the tenant/household currency is defined by the parent and propagated to all children and teens is the correct approach.
